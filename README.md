# Metamorfosis WebAR

## Περιγραφή

Η παρούσα εργασία είναι ένα απλό διαδραστικό εκπαιδευτικό WebAR demo βασισμένο στη **Μεταμόρφωση** του **Franz Kafka**.  
Η επιλογή του έργου έγινε στο πλαίσιο του μέρους Β, όπου είχε δοθεί ένας βαθμός ελευθερίας ως προς το υλικό.

Η *Μεταμόρφωση* γράφτηκε το 1912 και εκδόθηκε το 1915. Πρωταγωνιστής είναι ο Gregor Samsa, ένας νεαρός εμπορικός αντιπρόσωπος που στηρίζει οικονομικά την οικογένειά του. Ένα πρωί ξυπνά και ανακαλύπτει ότι έχει μεταμορφωθεί σε ένα τεράστιο έντομο. Από εκεί ξεκινά ένα έντονο δράμα που σχετίζεται με την απομόνωση, την αλλοτρίωση, την οικογένεια και την κοινωνική απόρριψη.

Στόχος της εφαρμογής δεν είναι να αντικαταστήσει το βιβλίο, αλλά να λειτουργήσει συμπληρωματικά στη διδασκαλία της λογοτεχνίας, δείχνοντας πώς η επαυξημένη πραγματικότητα μπορεί να δώσει ένα επιπλέον επίπεδο αλληλεπίδρασης.

## Τι κάνει η εφαρμογή

Η εφαρμογή παρουσιάζει ένα μικρό AR δείγμα από την πρώτη σκηνή του έργου, δηλαδή τη στιγμή που ο Gregor Samsa ξυπνά στο δωμάτιό του ήδη παραμορφωμένος.

Όταν ο χρήστης ανοίξει την εφαρμογή από κινητό και σαρώσει τον marker, εμφανίζεται πάνω στην εικόνα ένα AR κολάζ με βασικά στοιχεία του έργου. Η εμπειρία είναι απλή, αλλά προσπαθεί να δείξει πώς μπορεί ένα λογοτεχνικό κείμενο να συνδεθεί με εικόνα, ερμηνεία και διαδραστικότητα.

Στην παρούσα υλοποίηση υπάρχουν:

- τρεις διαφορετικοί «λογοτεχνικοί φακοί»: **Χώρος**, **Οικογένεια**, **Σώμα**
- εναλλαγή βασικής εικόνας/κολάζ
- εμφάνιση και απόκρυψη σύντομου αποσπάσματος
- hotspot με σύντομο ερμηνευτικό σχόλιο
- απλό ελληνικό περιβάλλον χειρισμού

## Γιατί επιλέχθηκαν τα A-Frame και MindAR

Για την υλοποίηση χρησιμοποιήθηκαν τα **A-Frame** και **MindAR**.

Το **A-Frame** χρησιμοποιήθηκε για να στηθεί η AR σκηνή μέσα από HTML, με σχετικά απλό τρόπο. Με αυτό ορίστηκαν τα βασικά στοιχεία της σκηνής, οι εικόνες, τα overlays και τα διαδραστικά στοιχεία.

Το **MindAR** χρησιμοποιήθηκε για την αναγνώριση του marker από την κάμερα του κινητού. Με απλά λόγια, αυτό είναι το κομμάτι που συνδέει την εικόνα-στόχο με την εμφάνιση του περιεχομένου επαυξημένης πραγματικότητας.

Άρα:

- το **A-Frame** στήνει τη σκηνή και τα οπτικά στοιχεία
- το **MindAR** αναλαμβάνει την ανίχνευση του marker

Οι επιλογές αυτές δεν έγιναν τυχαία. Ήταν οι πιο πρακτικές για να δημιουργηθεί ένα μικρό WebAR demo που να μπορεί να δοκιμαστεί σε πραγματική κινητή συσκευή.

## Παιδαγωγική λογική

Η εφαρμογή αντιμετωπίζεται ως συνοδευτικό μέσο διδασκαλίας και όχι ως υποκατάστατο του βιβλίου.

Το βιβλίο και το λογοτεχνικό κείμενο παραμένουν το βασικό σημείο αναφοράς. Η εφαρμογή όμως προσθέτει δυνατότητες που δεν υπάρχουν στην έντυπη μορφή, όπως:

- αναγνώριση marker
- εμφάνιση οπτικού υλικού πάνω στη σκηνή
- άμεση αλληλεπίδραση του μαθητή
- εναλλαγή διαφορετικών ερμηνευτικών οπτικών

Στη συγκεκριμένη περίπτωση ο μαθητής δεν βλέπει απλώς ένα στατικό κολάζ για τη *Μεταμόρφωση*, αλλά μπορεί να προσεγγίσει το υλικό μέσα από τρεις διαφορετικούς φακούς ανάγνωσης. Έτσι συνδέονται η λογοτεχνία, η εικόνα, ο ψηφιακός γραμματισμός και μια απλή μορφή διαδραστικής αφήγησης.

## Τι προστέθηκε πάνω στο αρχικό παράδειγμα

Η εργασία βασίστηκε αρχικά σε έτοιμο παράδειγμα **MindAR image tracking**, αλλά δεν έμεινε σε μια απλή εκτέλεση του demo.

Έγιναν ουσιαστικές αλλαγές, όπως:

- ελληνικό HUD διεπαφής
- κουμπιά ελέγχου
- λογική κατάστασης σε JavaScript
- εναλλαγή τριών σκηνών/κολάζ
- μηχανισμός «Λογοτεχνικού Φακού»
- hotspot με δυναμικό σχόλιο
- εμφάνιση/απόκρυψη αποσπάσματος
- responsive προσαρμογές για κινητό
- ξεχωριστή σελίδα marker για δοκιμή

Επομένως, η εργασία δεν είναι απλή χρήση ενός έτοιμου παραδείγματος, αλλά προσαρμογή του σε ένα συγκεκριμένο εκπαιδευτικό και λογοτεχνικό σενάριο.

## Εκτέλεση

Για να δουλέψει σωστά η εφαρμογή, χρειάζεται να τρέξει τοπικά από server και να ανοιχτεί από κινητό.

Παράδειγμα τοπικής εκτέλεσης:

```bash
cd ~/Gitprojects/metamorfosis
python3 -m http.server 8080
```

Έπειτα μπορεί να ανοιχτεί από browser στη διεύθυνση:

`http://localhost:8080`

Για δοκιμή σε κινητό, συνήθως χρειάζεται πρόσβαση από το ίδιο δίκτυο ή χρήση tunnel υπηρεσίας ώστε να δοθεί **ασφαλές https link**, γιατί η κάμερα του κινητού δεν λειτουργεί σωστά σε μη ασφαλές περιβάλλον.

## Αρχεία έργου

- `index.html` : κύρια AR σκηνή
- `js/app.js` : λογική αλληλεπίδρασης
- `styles/main.css` : μορφοποίηση και responsive προσαρμογές
- `marker.html` : βοηθητική σελίδα marker για δοκιμές
- `assets/images/` : βασικά εικαστικά στοιχεία
- `docs/changes-from-example.md` : συνοπτική καταγραφή αλλαγών πάνω στο αρχικό example
